# Agent Workspace Rules

- Start by reading `BUILDER.md` and `OPERATING.md`, then run `neon blueprint`.
- Keep actions approval-first.
- Keep secrets in environment variables.
- Do not commit `.env`.
- Do not run destructive account actions without explicit human approval.
- Prefer small, auditable changes.
- Use `neon memory add/search` for owner-created context only.
- For Discord or chat-channel behavior, follow `channels/discord-operating-floor.md`.
